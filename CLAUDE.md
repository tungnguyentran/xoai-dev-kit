## Overview

XoaiUtility is a multi-platform SwiftUI app (macOS / iOS / visionOS, deployment target 26.5, Swift 5.0). Despite the SwiftData scaffolding generated by the Xcode template, the app's only real feature is a **JSON formatter/viewer** (`JSONFormatterView`), which `ContentView` renders directly. The UI is designed for macOS (uses `HSplitView`, `NSPasteboard`, `NSColor`, `.command` keyboard shortcuts).

## Build & Test

There is no `.xcworkspace` for a Swift package — use the Xcode project. The scheme is `XoaiUtility`.

```bash
# Build (pick a destination matching SUPPORTED_PLATFORMS: macosx, iphonesimulator, xrsimulator)
xcodebuild -project XoaiUtility.xcodeproj -scheme XoaiUtility -destination 'platform=macOS' build

# Run all tests (unit + UI)
xcodebuild -project XoaiUtility.xcodeproj -scheme XoaiUtility -destination 'platform=macOS' test

# Run a single test (Swift Testing, not XCTest, for unit tests)
xcodebuild test -project XoaiUtility.xcodeproj -scheme XoaiUtility -destination 'platform=macOS' \
  -only-testing:XoaiUtilityTests/XoaiUtilityTests/example
```

Unit tests (`XoaiUtilityTests`) use the **Swift Testing** framework (`import Testing`, `@Test`, `#expect`) — not XCTest. UI tests (`XoaiUtilityUITests`) use XCTest. Both are currently empty scaffolding.

## Architecture

- **`JSONFormatterView.swift`** holds essentially all application logic:
  - `JSONNode` — an `indirect enum` modeling parsed JSON (object/array/string/number/bool/null). Built from `JSONSerialization`'s `Any` output via `JSONNode.build(from:sortKeys:)`. Numbers are stored as `String` (formatted via `NumberFormatter`) to preserve precision; booleans are distinguished from numbers by checking `CFGetTypeID(number) == CFBooleanGetTypeID()`.
  - `JSONFormatterView` — the editor screen. `process(pretty:)` is the core path used by both Format (⌘F) and Minify (⌘M): it parses with `JSONSerialization` (`.fragmentsAllowed`), re-serializes for the **Text** output, and builds a `JSONNode` for the **Tree** output. Errors surface in the status bar rather than throwing.
  - `JSONNodeView` — recursive SwiftUI view rendering the collapsible tree.

- **Indentation is post-processed**, not native. `JSONSerialization` only emits 2-space pretty output, so `reindent(_:indent:)` rewrites leading whitespace per line (every 2 leading spaces = one level → the chosen 2/4/tab unit). Changing indent options means working through this function.

- **Expand/Collapse All** uses a counter (`expansionVersion`) bumped on each press; each `JSONNodeView` watches it via `.onChange` and resets `isExpanded` to `defaultExpanded`. This is the mechanism for forcing a global expansion state across the independently-stateful recursive nodes.

- **`Item.swift` / SwiftData** (`ModelContainer` in `XoaiUtilityApp`) are leftover template boilerplate — `Item` is not used by any view. Don't assume persistence is part of the feature set.
